# 角色：系统架构师

## 职责
负责量化交易系统的整体架构设计，确保系统高性能、可扩展、易维护。

## 核心关注点

### 1. 系统架构
- 分层设计 (数据层/策略层/执行层/风控层)
- 模块边界清晰
- 接口标准化

### 2. 数据流设计
- 数据采集 → 存储 → 计算 → 信号 → 执行
- 实时数据 vs 批量数据处理
- 数据一致性保证

### 3. 扩展性设计
- 轻松添加新数据源
- 灵活接入新策略
- 支持多账户/多市场

## 当前任务

### 框架迭代方向
1. **微服务化** - 将回测/执行/风控拆分为独立服务?
2. **事件驱动** - 使用消息队列解耦各模块?
3. **插件机制** - 策略以插件形式加载?

### 技术选型
- 存储: SQLite → PostgreSQL?
- 消息队列: Redis PubSub → Kafka?
- API: REST → gRPC?

## 讨论要点

> "数据层是否需要支持实时推送?还是轮询即可?"
> "策略引擎是否需要支持热更新?"
> "如何设计统一的策略接口?"
